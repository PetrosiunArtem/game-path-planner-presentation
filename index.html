Cuphead Path Planner

Персональные маршруты к целям + оценка времени

MVP для одной игры — Cuphead

Автор: @PetrosiunArtem

Питч: строим лучший порядок уровней/покупок/практик под навыки игрока, показываем альтернативы и ETA.

Описание идеи

Какую проблему решаем?

  • Гайды универсальны, но не учитывают ваши навыки, оружие и прогресс.
  • Игроку сложно оценить, сколько времени займет цель и какой порядок шагов лучше.

Основной функционал

  • Выбор цели (S‑Rank, Pacifist, все оружия до Isle III).
  • Ввод профиля: оружия/чары/супер‑арты + навыковый профиль (точность, парри, выживаемость, движение, знание уровней).
  • Построение 2–3 альтернативных маршрутов с ETA и шансом успеха.

Чем лучше существующих?

  • Персонализация под навыки и текущий лоадаут.
  • Прозрачные расчеты времени и вероятностей, объяснение выбора маршрута.
  • Фокус на Cuphead: цели и метрики релевантны (S‑Rank критерии, Pacifist, монеты, DPS по фазам).

Пользовательские сценарии

  1. Все оружия до Isle III
    Игрок выбирает цель → отмечает пройденные Run‑and‑Gun/монеты → план получает оптимальный порядок монет и покупок (Roundabout/Spread/Charge).
  2. S‑Rank на Isle I
    Вводит навыки → получает рекомендованный лоадаут + план практик по парри → порядок боссов с ETA и кол-вом попыток.
  3. Pacifist на Run‑and‑Gun
    Получает маршрут, ключевые тайминги парри и точки риска → после каждой попытки обновляет прогресс, план пересчитывается.

Описание API (REST)

Base: /api/v1

  • GET /game/cuphead/goals — список целей
  • POST /game/cuphead/plan — расчёт плана по профилю
  • PATCH /profiles/{id}/progress — прогресс уровней/покупок
  • POST /telemetry/boss_attempt — факт попытки босса
  • POST /telemetry/run_level — факт прохождения Run‑and‑Gun
  • GET /content/version — версия контента

Модель времени и попыток

  • Босс: фазы HP / эффективный DPS (оружие × accuracy) + оверхед.
  • psuccess = sigmoid(a0 + a1·survival + a2·parry + a3·movement + a4·familiarity − a5·difficulty).
  • Ожидаемое: (T_raw + θ·T_raw·(1 − p)) / p, где θ≈0.5 — среднее время до смерти.
  • Run‑and‑Gun: T = T_base·(1 − movement·γ)·(1 − familiarity·δ) / p_success.

Безопасность и ToS

  • Без чтения памяти/инжекта. Ручной ввод/файл сейва (по желанию, с согласия).
  • Источник данных: публичные вики и speedrun‑метрики.
  • Телеметрия по времени — опт‑ин, минимум персональных данных.

Бизнес‑модель и метрики

  • Freemium: базовый план бесплатно; PRO — «советник лоадаута», тренировки и аналитика.
  • Метрики: точность ETA (MAPE), доля завершений планов, снижение попыток, D7 ретеншн.

План MVP (2 недели)

  1. Сбор HP/фаз боссов, базового DPS оружий, маршрутов монет.
  2. JSON‑граф 20–40 действий, оценщик времени/попыток.
  3. Планировщик A* + Монте‑Карло; 2–3 альтернативы.
  4. UI: профиль → план → альтернативы → чеклист → пересчёт.
