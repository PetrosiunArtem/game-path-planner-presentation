Cuphead Path Planner

Персональные маршруты к игровым целям + точная оценка времени

MVP для одной игры — Cuphead

Автор: @PetrosiunArtem

Питч: помогаем игроку Cuphead выбирать оптимальные шаги: какие уровни/боссы/покупки и в каком порядке выполнить, чтобы достичь цели (S-Rank, Pacifist, все оружия), прогнозируя время с учётом навыков.

Идея и ценность

  • Проблема: гайды по Cuphead общие. Игрок не понимает, сколько времени займёт цель и какой порядок даст лучший шанс с его навыками.
  • Решение: планировщик прогресса для Cuphead, который строит персональный маршрут и даёт 2–3 альтернативы.
  • Особенность: вместо «уровня героя» учитываем навыковый профиль (точность, парри, выживаемость, движение, знание конкретных уровней).

Почему мы лучше гайдов

  • Персонализация под оружие/чары/супер-арты и умения игрока.
  • Прозрачная оценка времени и вероятности успеха по каждому шагу.
  • Объяснимость: «почему именно этот путь?» — вклад шагов в общую экономию времени.

Пользовательские сценарии (2–3)

  1. Все оружия до Isle III: собрать монеты → купить ключевые пушки (Roundabout/Spread/Charge) → рекомендованный порядок уровней.
  2. S-Rank на Isle I: подобрать лоадаут → план тренировок парри → порядок боссов с наилучшим ETA и шансом успеха.
  3. Pacifist на Run-and-Gun: маршруты, тайминги парри, когда использовать Smoke Bomb; чеклист и пересчёт после неудач.

REST API (черновик)

Base URL: /api/v1

  • GET /game/cuphead/goals — цели (S-Rank, Pacifist, оружия)
  • POST /game/cuphead/plan — расчёт плана по профилю
  • PATCH /profiles/{id}/progress — обновление статуса уровней
  • POST /telemetry/boss_attempt — факт попытки босса
  • POST /telemetry/run_level — факт прохождения Run-and-Gun
  • GET /content/version — версия контента

Модель времени и попыток

  • Время босса = сумма фаз HP / эффективный DPS (зависит от оружия и accuracy) + оверхед.
  • Вероятность успеха попытки psuccess = sigmoid(a0 + a1*survival + a2*parry + a3*movement + a4*familiarity − a5*difficulty).
  • Ожидаемое время: (T_raw + θ*T_raw*(1−p)) / p, где θ≈0.5 — среднее время до смерти.
  • Run-and-Gun: T = T_base*(1 − movement*γ)*(1 − familiarity*δ) / p_success.

Безопасность и ToS

  • Без чтения памяти и вмешательства — только ручной ввод/файлы сейвов (опционально, по согласию).
  • Публичные источники: Fandom Wiki, speedrun.com.
  • Конфиденциальность: телеметрия по времени — опт-ин.

Бизнес‑модель и метрики

  • Freemium: базовый план бесплатно; PRO — «советник лоадаута», тренировки и расширенная аналитика.
  • Метрики: точность ETA (MAPE), доля завершений планов, снижение числа попыток.

План MVP (2 недели)

  1. Сбор данных по боссам/уровням/оружию (HP, DPS, coin routes).
  2. JSON‑граф на 20–40 действий и оценщик времени.
  3. Планировщик A* + симуляция попыток (Монте‑Карло).
  4. UI: профиль → план → альтернативы → чеклист.
